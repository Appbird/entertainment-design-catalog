### Experience Design Catalog
EDCはエンタテインメント領域において有用とされるデザインを、デザインの効果とそのアプローチによって特徴づけ、集積・分類を行うためのフレームワークである。
フレームワークに当てはめることによって得られた一つのデザイン典型例を「Archetype」と呼び、
このArchetypeを集積することで一つのCatalogを作るのが目的といえよう。

EDCでは、次の要件に対して、これらのガイドラインを定める。

1. 文脈
- どのような文脈を持ったユーザが、論文中で提案されているデザインに触れ始めるか？
- ここでいうユーザは、エンタテインメント的な価値を持つ体験を享受する対象に限定する。
	- 例えば、舞台を補助するようなツールでは、エンタテインメント体験を享受するのは観客であって、演者や裏方の人ではない。
	- 論文の述べるメインターゲットが存在しても、それが今回いうユーザとは限らない。
- ペルソナを新たに提案することもできる。この場合、次のようなユーザの特性を参考にすると良い。
	- Maslowの欲求理論の「生理的欲求」「安全の欲求」「社会的欲求」「承認欲求」「自己実現の欲求」
	- Self-determination theoryの「Autonomy、Competence、Relatedness」
	- Roger Cailloisの遊びの四要素「競争・偶然・模倣・めまい」

2. アプローチ
- コンテンツは、文脈で言及したそれぞれのユーザに対して、どのような体験を提供するか？
	- 2-1. 論文で主張されているデザインによって提供される体験について論ぜよ。
	- 2-2. 提案システムが元からあったコンテンツを継承して作られている場合、その継承した部分によって提供されているデザインの効果も論ぜよ。
		- 継承していない場合は、この設問は無視する。
- 体験は以下の過程に沿う形で記述し、それぞれのレイヤーでシステムによってどのような効果がもたらされるかについて整理せよ。
- 知覚
	- コンテンツから情報を受け取り、物体の空間的位置やその物理的動作を表す情報を得る。
	- ただし、文脈にユーザが影響され、コンテンツの表現と知覚にズレが生じる可能性がある。
	- 例：水面・木などの環境物の表現・物体の表現など。3DCGやイラストレーションで重要視される。
- 認知
	- 知覚と文脈をもとに、ユーザは何を解釈/錯覚するか？本来起きていないことで良い。
	- 正確に言えば、知覚・文脈をもとに、コンテンツの表現する世界に対して解釈・評価・予測する。
	- 例：「自分は大冒険をしている！」「キャラクターがかわいい」
- 情動
	- 知覚・文脈・認知をもとに、精神的反応（快/不快, 覚醒など）が生起される。
	- 例：「嬉しい」「ノスタルジーを覚える」
- 動機づけ
	- 上記をもとに、次にプレイヤが起こしたい行動について意思決定する。
	- 例：「運動したくなる」「ついつい見てしまう」「自慢したくなる」

3. デザイン効果タグ
- 上記で挙げられた効果に対して、ユーザにとって特に体験に寄与する効果を選び出せ。
	- 開発者が一番押し出したい体験かどうかを考え、一番上からその傾向が強いものを置いていく。
- ピックアップした効果について以下が読み取れるタイトルを設定せよ。
	- Type: そのデザインはユーザの(知覚-情報整理)と(情動-動機づけ)のどこに効果を生み出すか。
	- Effect: その部分において、どのような影響をデザインは与えているか。
    - Context: どういった文脈を持つユーザが、そのデザインに触れるか。
- Typeが知覚または解釈である場合は、その内容のみを説明せよ。
- Typeが情動または動機づけである場合、その対象となるシステムの表現（知覚-解釈）もEffectに含める必要がある。

以上に基づいて、この論文のArchetypeを整理せよ。

### 出力形式
以上の1. 2.についてコードブロック外で考察を与えよ。
その上で、3.について、次のような型を持つJSON in code blockに出力せよ。

[注意]
- 論文がユーザのエンタテインメント的な体験・ユーザの情動への貢献について言及していない場合、"design-tags"は[]とする。
- デザイン効果は一タグにつき一つまで。複数の効果について述べている場合は分割する。
- そのシステム(コンテンツ)内で固有の意味づけ(一般的ではない定義)を持つものや固有名詞を用いずに記述する.
	- e.g. Minecraftなら
     - レッドストーン → 論理回路
     - 黒曜石, 岩盤: 一般的な"黒曜石","岩盤"とは異なる意味づけ → 入手•達成困難な要素, プレイ範囲限界
     - 村人: 一般的な"村人"とは異なる意味づけ → 売買要素, NPC

```json
{
    "design-tags": {
		   "title": String, // 日本語で。タイトルは客観的ベースで語るよりも、主観的で妄想に寄り添ったタイトルの方がより良い。ただ、美しい〜とか驚きの〜とか、あまりにも美化しすぎないように。
           "type": String,  // "perception" | "cognition" | "emotion" |  "motivation"のいずれかでなくてはならない。
           "context": String, // 1.であげたユーザをここに記述する。
           "effect": String,
    }[]
}
```
